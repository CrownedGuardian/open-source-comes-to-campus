This page contains links to all of the curricula used at Open Source Comes to Campus events. Its main use is as a source for event planners to browse, compare, and select activities for their workshops. Overwhelmed by the options or not sure how these activities fit together? Check out the sample schedules.

Attendees of specific events may find the event-specific schedules link useful.

Some activities have multiple versions, usually self-guided or presenter-led. Some activities are specifically designed for small mentored groups but typically, if your event is using small mentored groups, they will use self-guided or presenter-led activities based on the particular mentor's comfort level.

To add new activites, please leave a comment in the issue tracker, contact hello@openhatch.org and/or submit your changes as a pull request.

  • Laptop Setup

    Prepares laptop for later activities.

    Typically, this takes about 15-20 minutes when done in person, though most attendees finish quickly. Can be done before an event to save time. Requires internet connection.

    Links: self-guided (bit.ly/laptop-setup); can be done in small groups or with a single presenter as well, by walking through the self-guided steps together.

  • What is Open Source?

    Defines open source, talks about popular projects, highlights FLOSS values.

    Takes approximately 20 minutes. Presenter-led version can be done without internet if downloaded ahead of time.

    Links: current version is presenter-led only and connected to tools: here.

  • What is version control?

    Introduces version control as a concept, using Wikipedia as an example.

    Typically takes about 10-15 minutes.

    Links: current version is presenter-led only and connected to other tools & intro to open source: here.

  • Diff and Patch

    Introduces diffing and patching as the building blocks of version control. Not a part of the standard curriculum, but interesting if you have the time!

    Links: self-guided, presenter-led.

  • Git

    There are many options for learning how to use git & github. There are currently three OpenHatch-created options: a self-guided version (teaches git only), a small mentored group version and a presenter-led version. Of these, we recommend the small mentored group version.

    There are a number of online curricula for teaching git which you may also want to consider: (list here).

    We currently have a presenter-led version which we're not terribly satisfied with. We're working on improving it and creating a self-guided version as well.

  • Career Panel

    Career panels are an excellent opportunity for students to learn more about what being an open source professional is like. If there are enough in person mentors we recommend running an in-person panel. We can also arrange a remote career panel via hangouts (or a similar tool).

    Links: you can read more about our career panels, including questions to ask, here.
